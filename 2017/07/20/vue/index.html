什么是Vue

Vue.js是一套构建用户界面的渐进式框架。与其他重量级框架不同的是，Vue 采用自底向上增量开发的设计。Vue 的核心库只关注视图层，它不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与单文件组件和 Vue 生态系统支持的库结合使用时，Vue 也完全能够为复杂的单页应用程序提供驱动。

Vue 使用

npm install vue
//引入并使用vue

声明式渲染

Vue.js 的核心是一个允许采用简洁的模板语法来声明式的将数据渲染进 DOM：

<div id="app">
  {{ message }}
</div>
var app = new Vue({
  el: '#app',
  data: {
    message: 'Hello Vue!'
  }
    })

模版语法

<span>Message: {{ msg }}</span>
双大括号里msg显示的是对应数据对象上msg的值
var app = new Vue({
  el: '#app',
  data: {
    msg: 'Hello Vue!'
      }
  })

指令

  • v-html这个 div 的内容将会被替换成为属性值 rawHtml，直接作为 HTML —— 数据绑定会被忽略

    <div v-html="rawHtml"></div>

  • v-if 指令将根据表达式 seen 的值的真假来移除/插入 <p> 元素。

    <p v-if="seen">Now you see me</p>

  • v-bind 指令被用来响应地更新 HTML 属性

    <!-- 完整语法 -->
<a v-bind:href="url"></a>
<!-- 缩写 -->
<a :href="url"></a>

  • v-on 指令，它用于监听 DOM 事件：

    <!-- 完整语法 -->
<a v-on:click="doSomething">
<!-- 缩写 -->
<a @click="doSomething"></a>

  • 修饰符（Modifiers）是以半角句号 . 指明的特殊后缀，用于指出一个指令应该以特殊方式绑定。例如，.prevent 修饰符告诉 v-on 指令对于触发的事件调用 event.preventDefault()：

    <form v-on:submit.prevent="onSubmit"></form>

  • v-for 指令根据一组数组的选项列表进行渲染，

    <ul id="example-1">
  <li v-for="item in items">
    {{ item.message }}
  </li>
</ul>
var example1 = new Vue({
  el: '#example-1',
  data: {
    items: [
      {message: 'Foo' },
      {message: 'Bar' }
    ]
  }
})

父子组件传值

  • 父组件 ==> 子组件

    //父组件先引入子组件所在的vue文件,
import Child from './Child.vue'

//再通过components引入子组件
  export default{
  name:'app',
  data(){
    return{
      citys:['北京','上海','天津','重庆']
    }
  },
  components:{ Child }  
  }

//在<template>中合适的位置放入Child,并传入值
<template>
<Child :citys='citys'></Child>
</template>

--------------------------------------------
//子组件通过props属性来过去父组件传入的值
props:{
    citys:Array //设置好citys对应的数据类型
}

  • 子组件 ==> 父组件

    //父组件自定义函数
<template>
  <Child @btnClick='clickHandler'></Child>
</template>
//子组件通过 this.$emit 拿到父组件自定义函数
liClick(){
  this.$emit('btnClick','Change Parent`s msg')
  // 第一个参数是触发 父级自定义的函数名
  // 第二个参数 数据
}
